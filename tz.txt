Техническое задание
Проект: Простой веб-мессенджер (аналог Telegram)
1. Общая цель

Создать простой веб-мессенджер с базовым функционалом общения пользователей в реальном времени, который можно будет расположить на github pages.

2. Технологические требования

Frontend:

Веб-приложение (SPA)

Современный стек (React / Vue / Svelte — на усмотрение)

Адаптивный интерфейс

Backend / BaaS:

Использовать Supabase (бесплатный тариф)

База данных: PostgreSQL (через Supabase)

Аутентификация — собственная логика через БД

Realtime-обновления через Supabase Realtime

3. Функционал пользователей
3.1 Регистрация

Пользователь может создать аккаунт.

Поля регистрации:

Никнейм (уникальный)

Пароль

Требования:

Проверка уникальности никнейма

Пароль хранить в хэшированном виде

Автоматический вход после регистрации

3.2 Авторизация

Пользователь может войти в систему.

Поля входа:

Никнейм

Пароль

3.3 Профиль пользователя

У каждого пользователя есть:

Никнейм

Аватарка (emoji на выбор)

Требования к аватарке:

Выбор одного emoji из встроенного списка

Emoji сохраняется в профиле пользователя

4. Чат
4.1 Общий чат

Все пользователи видят единый чат.

Возможности:

Отправка сообщений

Получение сообщений в реальном времени

Отображение:

Никнейма

Emoji-аватарки

Текста сообщения

Времени отправки

4.2 Ответы на сообщения

Реализовать систему ответов (reply).

Функционал:

Возможность ответить на конкретное сообщение

При ответе отображается:

На какое сообщение дан ответ

Никнейм автора исходного сообщения

Превью текста исходного сообщения

Визуально ответ должен быть вложенным

5. Интерфейс
5.1 Экран регистрации / входа

Переключение между формами входа и регистрации

Минималистичный интерфейс

5.2 Экран чата

Содержит:

Шапка приложения

Область сообщений

Поле ввода сообщения

Кнопка отправки

6. Структура базы данных (Supabase)
Таблица users

id (uuid, primary key)

nickname (text, unique)

password_hash (text)

avatar_emoji (text)

created_at (timestamp)

Таблица messages

id (uuid, primary key)

user_id (uuid, foreign key → users.id)

text (text)

reply_to (uuid, nullable, foreign key → messages.id)

created_at (timestamp)

7. Логика работы

Сообщения подгружаются автоматически (realtime)

Новые сообщения появляются без перезагрузки страницы

Пользователь видит, кто отправил сообщение

Ответы визуально связаны с исходными сообщениями

8. Минимальные требования (MVP)

Обязательно реализовать:

✅ Регистрация по никнейму и паролю
✅ Вход в систему
✅ Выбор emoji-аватарки
✅ Общий чат
✅ Отправка сообщений
✅ Ответы на сообщения
✅ Realtime-обновления
